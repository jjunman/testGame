# 밴드 합주 관리 앱 목록형 상세 개요

1. 앱 시작 (계정)

   a. 앱을 처음 실행하면 로그인 여부를 확인함

      i. 로그인된 계정이 없으면 로그인 화면으로 이동

      ii. 이미 로그인된 계정이 있으면 앱 메인 홈인 `내 밴드` 화면으로 이동

   b. 로그인 화면이 있음

      i. 이메일 입력칸, 비밀번호 입력칸, 로그인 버튼이 있음

      ii. 이메일 또는 비밀번호가 비어 있으면 로그인을 시도할 수 없거나 오류 문구가 표시됨

      iii. 로그인 성공 시 앱 메인 홈으로 넘어감

      iv. 로그인 실패 시 비밀번호 오류, 존재하지 않는 계정 등 실패 이유를 안내함

   c. 회원가입 화면이 있음

      i. 이름, 이메일, 비밀번호를 입력해 계정을 만들 수 있음

      ii. 계정 생성 후 로그인 화면으로 돌아가거나 로그인 상태로 앱 메인 홈에 진입함

      iii. 이미 사용 중인 이메일이거나 필수 입력값이 비어 있으면 오류를 보여줌

2. 앱 메인 홈 (`내 밴드`)

   a. 사용자가 가입한 밴드 목록이 표시됨

      i. 각 밴드는 카드 형태로 표시됨

      ii. 카드 배경에는 밴드 썸네일 이미지가 들어감

      iii. 카드에는 밴드 이름, 초대코드, 내 역할, 내 포지션, 멤버 수가 표시됨

      iv. 현재 선택된 밴드는 선택 상태 배지나 강조 테두리로 구분됨

   b. 밴드 추가 버튼이 있음

      i. `밴드 추가하기` 버튼을 누르면 밴드 추가 페이지로 이동함

      ii. 밴드가 하나도 없으면 “아직 가입한 밴드가 없음”과 같은 빈 상태 안내를 보여줌

   c. 프로필 이동 버튼이 있음

      i. 화면 상단의 사람 아이콘을 누르면 유저/프로필 탭으로 이동함

      ii. 프로필 화면에서는 로그인한 사용자 정보와 계정 관련 기능을 확인할 수 있음

3. 밴드 추가

   a. 밴드 추가 페이지로 이동함

      i. 앱 메인 홈에서 `밴드 추가하기` 버튼을 누르면 이동함

      ii. 화면 안에는 `밴드 만들기` 버튼과 초대코드 입력 영역이 있음

   b. 밴드 만들기 버튼이 있음

      i. `밴드 만들기` 버튼을 누르면 밴드 만들기 페이지로 이동함

      ii. 새 밴드를 직접 생성하려는 사용자가 사용하는 버튼임

   c. 초대코드 입력 공간이 있음

      i. 이미 만들어진 밴드에 가입하려면 초대코드를 입력함

      ii. 초대코드는 입력 시 대문자로 변환됨

      iii. 초대코드를 입력한 뒤 내 포지션을 선택하면 가입할 수 있음

      iv. 초대코드가 틀리면 해당 밴드를 찾을 수 없다는 오류가 표시됨

      v. 이미 가입한 밴드의 초대코드를 입력하면 이미 가입한 밴드라는 오류가 표시됨

4. 밴드 만들기

   a. 상단에서 밴드 썸네일을 설정할 수 있음

      i. 기본 썸네일 이미지가 먼저 표시됨

      ii. `갤러리에서 선택` 버튼을 누르면 휴대폰 사진 보관함에서 이미지를 선택할 수 있음

      iii. 사진 접근 권한이 없으면 권한이 필요하다는 안내가 뜸

      iv. 선택한 이미지는 밴드 대표 이미지로 저장됨

   b. 밴드 이름을 입력하는 input이 있음

      i. 밴드 이름은 최소 2자 이상 입력해야 함

      ii. 밴드 이름을 입력하지 않거나 너무 짧으면 오류 문구가 표시됨

      iii. 밴드 이름은 밴드 목록 카드, 밴드 홈 상단, 알림 제목 등에 사용됨

   c. 포지션을 정하는 버튼이 있음

      i. 선택 가능한 기본 포지션은 리드기타, 서브기타, 베이스, 드럼, 피아노, 보컬임

      ii. `직접 입력`을 선택하면 직접 포지션명을 작성하는 input이 표시됨

      iii. 직접 입력을 선택했는데 포지션명을 적지 않으면 밴드를 만들 수 없음

      iv. 선택한 포지션은 밴드 멤버 정보와 밴드 카드에 표시됨

   d. 밴드 생성 시 초대코드가 자동 생성됨

      i. 초대코드는 6자리 영문 대문자와 숫자로 만들어짐

      ii. 생성된 초대코드는 밴드 카드와 밴드 홈에서 확인할 수 있음

      iii. 다른 사용자는 이 초대코드를 입력해 밴드에 가입함

   e. 밴드 생성자는 리더가 됨

      i. 리더는 밴드를 만든 사용자임

      ii. 리더는 시작 볼륨 포인트 25점을 받음

      iii. 일반 멤버는 초대코드로 가입할 때 시작 볼륨 포인트 20점을 받음

      iv. 리더는 합주곡 투표 종료, 합주 시간 제안, 합주실 확정, 연습 과제 마감 같은 주요 결정을 할 수 있음

5. 밴드 입장하기

   a. 앱 메인 홈에서 밴드 카드가 보임

      i. 카드에는 밴드 썸네일 이미지가 크게 보임

      ii. 썸네일 위에 밴드 이름과 초대코드가 표시됨

      iii. 내 역할이 리더인지 멤버인지 배지로 표시됨

      iv. 내 포지션과 멤버 수가 함께 표시됨

   b. 밴드 카드 클릭 시 밴드 화면으로 이동함

      i. 사용자가 밴드 썸네일 카드를 누르면 해당 밴드 홈으로 이동함

      ii. 이동하면서 현재 선택 밴드가 해당 밴드로 저장됨

      iii. 이후 밴드 내부 네비게이션은 선택된 밴드 ID를 기준으로 동작함

6. 밴드 내부 네비게이션

   a. 밴드에 들어가면 하단 네비게이션 바가 보임

      i. 밴드 내부 화면에서는 하단에 고정된 네비게이션이 표시됨

      ii. 탭은 `홈`, `노래`, `캘린더`, `합주실`, `멤버`로 구성됨

      iii. 현재 보고 있는 탭은 배경색과 아이콘 색으로 강조됨

   b. 홈 탭이 있음

      i. 밴드 이름, 내 역할, 내 포지션, 초대코드, 해야 할 일, 진행 상황을 요약함

      ii. 밴드 준비 상태를 한 화면에서 확인하는 대시보드 역할을 함

   c. 노래 탭이 있음

      i. 합주곡 후보 추가와 투표를 진행함

      ii. 확정된 합주곡 목록과 개인 연습 과제로 이동할 수 있음

   d. 캘린더 탭이 있음

      i. 내 가능한 시간을 등록함

      ii. 리더가 합주 시간을 제안하고 멤버들이 찬반 투표를 함

   e. 합주실 탭이 있음

      i. 내 위치를 등록함

      ii. 합주실 후보를 추가하고 투표함

      iii. 리더가 최종 합주실을 확정함

   f. 멤버 탭이 있음

      i. 밴드 멤버 목록을 확인함

      ii. 각 멤버의 포지션, 역할, 볼륨 포인트를 확인함

7. 밴드 홈

   a. 상단에는 밴드 이름과 내 정보가 표시됨

      i. 밴드 이름이 가장 크게 표시됨

      ii. 내 포지션과 내 역할이 표시됨

      iii. 내가 리더인지 일반 멤버인지 확인할 수 있음

   b. 요약 정보가 표시됨

      i. 열려 있는 연습 과제 수가 표시됨

      ii. 등록된 일정 수가 표시됨

      iii. 밴드 초대코드가 표시됨

      iv. 해야 할 일이 있으면 필요한 작업 개수가 배지로 표시됨

   c. 내가 해야 할 것 목록이 있음

      i. 가장 중요한 할 일 1개는 큰 카드로 표시됨

      ii. 나머지 할 일은 작은 리스트로 표시됨

      iii. 할 일에는 합주곡 투표하기, 일정 등록하기, 합주 시간 찬반 투표하기, 합주실 투표하기, 연습 제출하기 등이 있음

      iv. 각 할 일 카드에는 제목, 설명, 마감 또는 상태 라벨, 바로가기 버튼이 있음

      v. 바로가기 버튼을 누르면 해당 기능 화면으로 이동함

   d. 투표 진행 현황이 있음

      i. 밴드 진행 타임라인 형태로 표시됨

      ii. `곡 정하기`, `연습하기`, `합주실 잡기` 단계가 표시됨

      iii. 완료된 단계는 체크 표시가 들어가고 초록색 완료 상태로 바뀜

      iv. 진행이 필요한 단계는 강조 색으로 표시됨

      v. 아직 시작되지 않은 단계는 대기 상태로 표시됨

   e. 우리 밴드 합주곡 목록과 연결됨

      i. 확정된 곡은 노래 탭의 `곡과 연습` 목록에 표시됨

      ii. 곡 목록에는 곡 썸네일, 제목, 가수, 연습 상태가 보임

      iii. 곡을 누르면 개인 연습 상세 화면으로 이동함

      iv. 진행 중인 합주곡 투표가 있으면 “합주곡 정하기 진행중” 카드처럼 표시될 수 있음

8. 합주곡 투표 시작

   a. 노래 탭에서 투표를 시작할 수 있음

      i. 하단 네비게이션에서 `노래`를 누르면 노래 탭으로 이동함

      ii. 노래 탭에는 `투표`와 `곡과 연습` 세그먼트가 있음

      iii. `투표` 세그먼트에서 합주곡 투표를 시작함

   b. 투표 시작 버튼이 있음

      i. 진행 중인 투표가 없으면 `투표 시작` 버튼이 표시됨

      ii. 이미 투표가 진행 중이면 `투표 진행 중` 상태가 표시됨

      iii. 투표가 진행 중일 때는 새 투표를 중복으로 시작할 수 없음

   c. 투표 시작 시 마감기한을 고름

      i. 투표 시작 버튼을 누르면 팝업이 뜸

      ii. 팝업에서 하루, 3일, 7일 중 하나를 선택함

      iii. 선택한 마감기한은 투표 라운드의 종료 시간으로 저장됨

      iv. 투표 화면에는 `D-남은 일수`가 표시됨

      v. 투표가 시작되면 멤버들에게 합주곡 투표 알림을 보낼 수 있음

9. 합주곡 후보 추가

   a. 투표 화면 하단 또는 액션 영역에 후보곡 추가 버튼이 있음

      i. `후보곡 추가` 버튼을 누르면 곡 추가 화면으로 이동함

      ii. 투표가 후보 모집 또는 투표 진행 상태일 때 후보곡을 추가할 수 있음

      iii. 완료된 투표 라운드에는 새 후보곡을 추가할 수 없음

   b. 사용자가 입력할 정보가 있음

      i. 곡 제목을 입력함

      ii. 가수 이름을 입력함

      iii. 유튜브 링크를 입력함

      iv. 유튜브 링크는 영상 ID를 추출할 수 있는 정상 링크여야 함

      v. 필요한 악기 정보가 있으면 곡 후보에 함께 저장될 수 있음

   c. 중복 곡을 막음

      i. 같은 투표 라운드 안에 같은 제목과 같은 가수의 곡이 있으면 추가할 수 없음

      ii. 중복이면 이미 추가된 곡이라는 오류가 표시됨

   d. 밴드 포지션과 곡 필요 악기를 비교함

      i. 후보곡에 필요한 악기 정보가 있으면 현재 밴드 멤버들의 포지션과 비교함

      ii. 밴드에 없는 파트가 필요한 곡이면 경고 문구가 표시됨

      iii. 예를 들어 우리 밴드에 피아노 멤버가 없는데 피아노가 필요한 곡이면 해당 파트가 부족하다고 알려줌

10. 합주곡 게시 및 투표 단계

   a. 화면 상단에는 투표 마감까지 남은 시간이 표시됨

      i. 마감일이 있으면 `D-?` 형태로 표시됨

      ii. 마감 날짜와 시간도 함께 확인할 수 있음

   b. 게시된 곡 목록이 카드처럼 보임

      i. 곡 후보 카드는 가로 스크롤로 볼 수 있음

      ii. 사용자는 좌우로 넘기면서 후보곡들을 살펴봄

      iii. 현재 보고 있는 카드에는 유튜브 플레이어가 표시됨

      iv. 현재 보지 않는 카드에는 유튜브 썸네일이 표시됨

   c. 각 카드에는 곡 정보가 있음

      i. 유튜브 플레이어 또는 썸네일이 있음

      ii. 곡 제목이 있음

      iii. 가수 이름이 있음

      iv. 현재 득표 수가 표시됨

      v. 투표하기 버튼이 있음

      vi. 내가 올린 후보라면 삭제 버튼이 표시될 수 있음

   d. 한 명당 최대 2곡까지 투표 가능함

      i. 마음에 드는 곡에 투표 버튼을 누름

      ii. 선택된 곡은 카드 테두리와 버튼 상태가 바뀜

      iii. 이미 선택한 곡을 다시 누르면 투표를 취소할 수 있음

      iv. 2곡을 이미 선택한 상태에서 다른 곡을 추가로 선택하면 최대 2곡 제한 안내가 뜸

   e. 제출하기 버튼이 있음

      i. 1곡 이상 선택하면 제출할 수 있음

      ii. 제출 버튼을 누르면 현재 선택 상태가 서버에 반영됨

      iii. 마감 전에는 선택지를 바꿀 수 있음

      iv. 마감 후에는 선택을 바꿀 수 없고 결과가 확정됨

11. 합주곡 투표 완료 단계

   a. 리더가 투표를 종료할 수 있음

      i. 리더에게는 `지금 끝내기` 버튼이 보임

      ii. 버튼을 누르면 현재 투표 결과로 합주곡을 확정할지 묻는 팝업이 뜸

      iii. 확인하면 투표가 종료되고 최종 곡이 확정됨

   b. 자동 또는 수동으로 최종 곡이 결정됨

      i. 마감기한이 끝나면 결과가 확정될 수 있음

      ii. 리더가 직접 종료하면 마감 전에도 확정할 수 있음

   c. 득표 결과 계산 방식이 있음

      i. 후보곡에 투표한 멤버들의 볼륨 포인트 합계를 먼저 비교함

      ii. 볼륨 포인트 합계가 높은 곡이 우선됨

      iii. 포인트 합계가 같으면 가장 높은 볼륨 포인트를 가진 투표자가 포함된 곡이 우선됨

      iv. 그래도 같으면 득표 수를 비교함

   d. 확정된 곡이 표시됨

      i. 최종 곡의 썸네일이 표시됨

      ii. 곡명이 표시됨

      iii. 가수명이 표시됨

      iv. 확정곡은 `곡과 연습` 목록에 추가됨

      v. 밴드 홈의 `곡 정하기` 단계가 완료 상태로 바뀜

12. 우리 밴드 합주곡 목록

   a. 노래 탭의 `곡과 연습` 영역에 표시됨

      i. 투표로 확정된 곡들이 목록으로 보임

      ii. 각 곡은 카드 또는 리스트 아이템 형태로 표시됨

      iii. 카드에는 곡 썸네일, 제목, 가수, 연습 상태가 있음

   b. 곡 카드 클릭 시 연습 화면으로 이동함

      i. 해당 곡에 연습 과제가 있으면 연습 상세 화면으로 이동함

      ii. 아직 연습 과제가 없으면 연습 과제 만들기 화면으로 이동할 수 있음

   c. 곡 관리 메뉴가 있음

      i. 곡 카드의 더보기 버튼을 누르면 수정, 삭제, 닫기 메뉴가 표시됨

      ii. 수정에서는 제목, 가수, 유튜브 링크를 바꿀 수 있음

      iii. 삭제하면 곡 목록에서 제거됨

      iv. 연결된 연습 과제가 있으면 과제 자체가 아니라 곡과 과제의 연결이 해제될 수 있음

13. 개인 연습 화면

   a. 상단에 곡 정보가 표시됨

      i. 곡 썸네일이 배경처럼 표시됨

      ii. 곡명과 가수가 표시됨

      iii. 마감일이 표시됨

      iv. 제출 완료 여부가 배지로 표시됨

   b. 과제-연습-제출-결과 타임라인이 있음

      i. `과제`, `연습`, `제출`, `결과` 네 단계가 있음

      ii. 각 단계는 아이콘과 텍스트로 표시됨

      iii. 완료된 단계는 체크 표시가 들어감

      iv. 현재 선택된 단계는 강조 색으로 표시됨

      v. 각 단계를 누르면 해당 영역으로 이동함

14. 개인 연습: 과제 단계

   a. 연습 과제 정보를 보여줌

      i. 과제 제목이 표시됨

      ii. 과제 설명이 표시됨

      iii. 마감일이 표시됨

      iv. 연습할 구간이 표시됨

   b. 연습 구간이 있음

      i. 곡의 몇 초부터 몇 초까지 연습할지 시작 초와 종료 초로 저장됨

      ii. 예를 들어 35초부터 80초까지처럼 구간을 정함

      iii. 화면에서는 0:35 - 1:20 같은 형식으로 표시됨

      iv. 구간이 정해져 있으면 녹음본 길이가 해당 구간보다 짧을 때 제출할 수 없음

   c. 리더는 과제를 마감할 수 있음

      i. 리더에게는 과제 마감 버튼이 보일 수 있음

      ii. 과제를 마감하면 제출 현황과 믹싱 결과를 확인하는 단계로 넘어감

15. 개인 연습: 연습 단계

   a. 유튜브 플레이어가 있음

      i. 확정곡의 유튜브 링크를 기반으로 플레이어를 띄움

      ii. 유튜브 영상 ID가 있으면 해당 영상을 재생함

      iii. 영상 ID를 찾지 못하면 썸네일 또는 대체 화면이 표시될 수 있음

   b. 녹음 버튼이 있음

      i. 녹음 버튼을 누르면 이어폰을 꼭 착용해 달라는 alert가 뜸

      ii. 마이크 권한이 없으면 권한 요청 화면 또는 안내가 표시됨

      iii. 권한이 허용되면 5초 카운트다운이 시작됨

   c. 5초 준비 시간이 있음

      i. 유저는 5초 동안 음원에 맞춰 들어갈 준비를 함

      ii. 카운트다운이 끝나면 실제 녹음이 시작됨

      iii. 녹음 중에는 파형 데이터가 기록됨

   d. 녹음을 끝내거나 취소할 수 있음

      i. 녹음 중 버튼을 다시 누르면 녹음이 종료됨

      ii. 종료한 녹음은 임시 녹음본으로 자동 저장됨

      iii. 취소 버튼을 누르면 녹음이 저장되지 않음

      iv. 녹음본은 최대 5개까지 임시 저장됨

   e. 오디오 파일을 직접 불러올 수 있음

      i. 사용자는 녹음 대신 오디오 파일을 선택할 수 있음

      ii. 선택한 파일도 임시 녹음본 목록에 추가됨

      iii. 과제 구간보다 길이가 짧으면 추가 또는 제출이 제한됨

16. 개인 연습: 제출 단계

   a. 저장된 녹음본 목록이 표시됨

      i. 각 녹음본은 생성 시간과 녹음 방식이 표시됨

      ii. 녹음본에는 파형 미리보기가 표시됨

      iii. 사용자는 녹음본을 선택할 수 있음

   b. 녹음본을 들어볼 수 있음

      i. 각 녹음본에는 재생 버튼이 있음

      ii. 재생 버튼을 누르면 해당 녹음본을 확인할 수 있음

      iii. 마음에 드는 녹음본을 선택해 제출함

   c. 제출하기 버튼이 있음

      i. 선택된 녹음본이 없으면 제출할 수 없음

      ii. 선택된 녹음본이 있으면 서버에 업로드함

      iii. 업로드가 성공하면 제출 완료 alert가 뜸

      iv. 제출 후 연습 상세 화면에서 내 제출 상태가 완료로 바뀜

17. 개인 연습: 결과 단계

   a. 내 제출본을 확인할 수 있음

      i. 내가 제출한 녹음본이 있으면 제출 시간이 표시됨

      ii. 제출본을 재생할 수 있음

   b. 멤버별 제출 현황을 확인할 수 있음

      i. 멤버 이름이 표시됨

      ii. 제출 여부가 표시됨

      iii. 제출 시간이 표시됨

      iv. 포인트 변화량이 표시됨

   c. 믹싱본을 들을 수 있음

      i. 제출본들이 있으면 리더가 믹싱본을 생성할 수 있음

      ii. 믹싱본이 생성되면 재생 버튼으로 들어볼 수 있음

      iii. 과제가 마감되었고 제출본이 있으면 자동 믹스 생성을 시도할 수 있음

18. 합주 스케줄러 메인

   a. 캘린더 탭에서 접근함

      i. 밴드 내부 네비게이션에서 `캘린더`를 누르면 이동함

      ii. 화면 상단에는 캘린더 배너가 있음

   b. 주요 버튼이 있음

      i. `내 일정 등록하기` 버튼이 있음

      ii. `일정 투표하기` 버튼이 있음

      iii. `합주 시간 제안하기` 버튼이 있음

      iv. 일정 투표하기는 진행 중인 제안이 있을 때만 활성화됨

      v. 합주 시간 제안하기는 리더가 사용하는 기능임

   c. 시간표 미리보기가 있음

      i. 오전 시간표와 오후 시간표가 나뉘어 표시됨

      ii. 요일별, 시간별 칸으로 구성됨

      iii. 내가 선택한 시간은 연한 색으로 표시됨

      iv. 모두가 가능한 시간은 진한 강조 색으로 표시됨

      v. 아무도 선택하지 않았거나 내가 선택하지 않은 시간은 빈 칸으로 표시됨

19. 내 일정 등록하기

   a. 시간 맞추기 화면으로 이동함

      i. 오전 또는 오후 시간대를 선택해 들어갈 수 있음

      ii. 화면에는 일주일치 요일과 시간 칸이 보임

   b. 가능한 시간을 선택함

      i. 사용자는 가능한 시간 칸을 눌러 선택함

      ii. 선택한 칸은 색이 바뀜

      iii. 다시 누르면 선택이 해제됨

      iv. 여러 칸을 선택할 수 있음

   c. 선택한 시간을 저장함

      i. `확인` 버튼을 누르면 선택 시간이 서버에 저장됨

      ii. 각 시간은 1시간 단위로 저장됨

      iii. 기존에 선택했다가 해제한 시간은 불가능 상태로 바뀜

      iv. 저장 후 스케줄러 메인으로 돌아감

20. 합주 시간 제안하기

   a. 리더가 합주 시간을 제안함

      i. `합주 시간 제안하기` 버튼을 누르면 제안 화면으로 이동함

      ii. 이 화면은 멤버들이 등록한 가능한 시간을 바탕으로 추천 시간을 보여줌

   b. 합주 길이를 선택함

      i. 1시간, 2시간, 3시간, 4시간 버튼이 있음

      ii. 리더가 원하는 합주 시간을 선택함

      iii. 선택한 길이에 맞는 연속 시간대를 추천함

   c. 모두가 가능한 시간을 추천함

      i. 모두 가능한 시간대를 가장 우선 추천함

      ii. 모두 가능한 시간이 여러 개면 가장 빠른 시간을 우선함

      iii. 모두 가능한 시간이 없으면 가능한 멤버 수가 많은 시간을 추천함

      iv. 추천 시간은 최대 3개까지 표시됨

   d. 시간을 직접 정할 수 있음

      i. 요일을 직접 선택할 수 있음

      ii. 시작 시간을 입력할 수 있음

      iii. 종료 시간을 입력할 수 있음

      iv. 시간 형식은 19:00 같은 형태로 입력함

      v. 종료 시간이 시작 시간보다 빠르면 오류가 표시됨

   e. 제안 완료 시 찬반 투표가 시작됨

      i. 제안하기 버튼을 누르면 합주 시간 찬반 투표가 생성됨

      ii. 멤버들의 밴드 홈 할 일 목록에 합주 시간 투표하기가 표시됨

      iii. 캘린더 메인 화면의 일정 투표하기 버튼이 활성화됨

21. 일정 찬반 투표

   a. 리더가 제안한 시간이 표시됨

      i. 제안된 요일이 표시됨

      ii. 시작 시간과 종료 시간이 표시됨

      iii. 찬성 수와 반대 수가 표시됨

      iv. 내 응답 상태가 표시됨

   b. 찬성 또는 반대를 선택함

      i. 사용자는 `찬성` 버튼을 누를 수 있음

      ii. 사용자는 `반대` 버튼을 누를 수 있음

      iii. 한 번 제출하면 변경할 수 없음

      iv. 이미 응답한 사용자는 버튼이 잠김

   c. 제출 버튼 아래에 안내 글씨가 뜸

      i. 모두가 가능한 시간대라면 “모두가 가능한 시간대예요”라고 표시됨

      ii. 일부 멤버가 불가능하면 “멤버 A, B를 제외하고 가능한 시간대예요”처럼 표시됨

      iii. 내가 불가능한 시간대라면 내가 불가능한 시간대라는 안내가 표시됨

   d. 리더가 투표를 종료할 수 있음

      i. 리더에게는 `지금 끝내기` 버튼이 보임

      ii. 모든 멤버가 찬성하면 합주 시간이 확정됨

      iii. 반대가 있으면 확정되지 않을 수 있음

      iv. 확정된 시간은 완료 카드로 표시됨

22. 합주실 정하기

   a. 합주실 탭으로 이동함

      i. 밴드 내부 네비게이션에서 `합주실`을 누르면 이동함

      ii. 상단에는 합주실 정하기 배너가 표시됨

   b. 내 위치를 정할 수 있음

      i. 현재 위치 버튼을 눌러 내 위치를 가져올 수 있음

      ii. 주소를 입력해 위치를 검색할 수 있음

      iii. 지도에서 직접 위치를 찍을 수 있음

      iv. 위치를 저장하면 합주실 후보와의 거리 계산에 사용됨

      v. 저장된 위치가 있으면 저장됨 배지가 표시됨

   c. 합주실 후보 추가 버튼이 있음

      i. `합주실 후보 추가` 버튼을 누르면 후보 추가 페이지로 이동함

      ii. 후보 추가 페이지에서는 저장된 합주실 목록을 볼 수 있음

      iii. 합주실은 평균 거리가 짧은 순서로 추천됨

      iv. 처음에는 일부 후보만 보이고 전체 보기를 통해 더 많이 볼 수 있음

   d. 합주실 후보 목록이 표시됨

      i. 후보는 가로 스크롤 카드로 표시됨

      ii. 카드에는 합주실 이름이 표시됨

      iii. 주소가 표시됨

      iv. 시간당 가격이 있으면 표시됨

      v. 평균 거리가 표시됨

      vi. 내 위치 기준 거리와 멤버 위치 누락 수가 표시될 수 있음

      vii. 추천 1순위 후보에는 추천 배지가 붙음

      viii. 전화 버튼이나 외부 링크 버튼이 있을 수 있음

   e. 후보를 골라 투표함

      i. 사용자는 마음에 드는 합주실 후보에 투표함

      ii. 내가 투표한 후보는 선택 상태로 강조됨

      iii. 후보 카드에는 현재 득표 수가 표시됨

   f. 리더가 합주실을 확정함

      i. 리더에게는 `합주실 확정하기` 버튼이 보임

      ii. 버튼을 누르면 현재 투표 결과로 합주실이 확정됨

      iii. 확정된 합주실은 별도 카드로 표시됨

      iv. 밴드 홈의 합주실 단계가 완료 상태로 바뀜

23. 합주실 후보 추가

   a. 합주실 목록을 보여줌

      i. 앱에 저장된 안산 지역 합주실 목록이 표시됨

      ii. 합주실 이름, 지역, 주소, 가격, 링크를 확인할 수 있음

      iii. 멤버 위치 정보가 있으면 평균 거리 기준으로 정렬됨

   b. 지도에서 위치를 확인할 수 있음

      i. 목록에서 합주실을 누르면 지도에서 위치를 볼 수 있음

      ii. 위도와 경도 정보가 있는 합주실은 지도 마커로 표시됨

      iii. 위치 정보가 없는 합주실은 거리 계산이 제한될 수 있음

   c. 후보로 추가할 수 있음

      i. 원하는 합주실을 선택해 후보로 추가함

      ii. 후보로 추가하면 합주실 탭 하단 후보 목록에 표시됨

      iii. 추가된 후보는 멤버들이 투표할 수 있음

24. 멤버 / 볼륨 포인트

   a. 멤버 탭에서 멤버 목록을 확인함

      i. 멤버 이름이 표시됨

      ii. 멤버 역할이 표시됨

      iii. 멤버 포지션이 표시됨

      iv. 가입일이 표시됨

      v. 볼륨 포인트가 표시됨

   b. 시작 포인트가 있음

      i. 리더는 25포인트를 받고 시작함

      ii. 일반 멤버는 20포인트를 받고 시작함

   c. 연습 제출에 따라 포인트가 변함

      i. 녹음을 제시간에 제출하면 볼륨 포인트가 상승함

      ii. 제출하지 않으면 볼륨 포인트가 하락함

      iii. 포인트 변화는 연습 결과 화면의 멤버별 상태에 표시됨

   d. 볼륨 포인트가 높은 사람의 특권이 있음

      i. 합주곡 투표에서 결과가 비슷할 때 볼륨 포인트가 높은 멤버들의 선택이 더 큰 영향력을 가짐

      ii. 같은 득표수 또는 비슷한 결과가 나오면 포인트 기준으로 우선순위를 정함

      iii. 합주 시간 정하기에서도 같은 득표수가 나오면 볼륨 포인트가 높은 사람의 선택이 우선될 수 있음

25. 전체 사용 흐름

   a. 새 사용자가 앱을 시작함

      i. 회원가입 또는 로그인을 함

      ii. 로그인 후 내 밴드 화면으로 이동함

   b. 밴드에 들어감

      i. 직접 밴드를 만들거나 초대코드로 밴드에 가입함

      ii. 밴드 카드에서 원하는 밴드를 눌러 밴드 홈으로 이동함

   c. 합주곡을 정함

      i. 리더가 합주곡 투표를 시작함

      ii. 멤버들이 후보곡을 추가함

      iii. 멤버들이 최대 2곡까지 투표함

      iv. 리더가 투표를 종료하거나 마감 시간이 지나면 곡이 확정됨

   d. 개인 연습을 진행함

      i. 확정곡으로 연습 과제를 만듦

      ii. 멤버들이 정해진 구간을 연습하고 녹음함

      iii. 녹음본 중 하나를 제출함

      iv. 결과 화면에서 제출 현황과 믹싱본을 확인함

   e. 합주 시간을 정함

      i. 멤버들이 가능한 시간을 등록함

      ii. 리더가 추천 시간 또는 직접 입력으로 합주 시간을 제안함

      iii. 멤버들이 찬성 또는 반대 투표를 함

      iv. 조건이 맞으면 합주 시간이 확정됨

   f. 합주실을 정함

      i. 멤버들이 위치를 저장함

      ii. 합주실 후보를 추가함

      iii. 멤버들이 후보에 투표함

      iv. 리더가 최종 합주실을 확정함

   g. 밴드 준비 과정이 완료됨

      i. 합주곡, 개인 연습, 합주 시간, 합주실이 모두 정리됨

      ii. 밴드 홈에서 각 단계가 완료 상태로 표시됨

      iii. 멤버별 볼륨 포인트와 제출 결과가 남아 다음 활동에 반영됨
